This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Structure

This is a **pnpm + Turborepo monorepo** with two apps and one shared package:

- `apps/api` — Fastify REST API (Node.js, TypeScript, Prisma, PostgreSQL)
- `apps/web` — Next.js 16 frontend (App Router, React 19, Tailwind CSS v4, shadcn/ui)
- `packages/typescript-config` — Shared TypeScript configs (`nextjs.json`, `nodejs.json`)

## Commands

### Root (runs across all workspaces via Turbo)
```bash
pnpm dev          # Start all apps in dev mode
pnpm build        # Build all apps
pnpm lint         # Run Biome linter on all apps
pnpm format       # Run Biome formatter on all apps
pnpm check-types  # TypeScript type checking on all apps
```

### API (`apps/api`)
```bash
pnpm dev                    # tsx watch mode with .env loaded
pnpm build                  # tsc compile (runs prisma migrate deploy && prisma generate first)
pnpm prisma migrate dev     # Create and run a new migration
pnpm prisma generate        # Regenerate Prisma client to src/generated/prisma/
pnpm prisma studio          # Open Prisma Studio GUI
pnpm lint                   # Biome check
```

### Web (`apps/web`)
```bash
pnpm dev          # Next.js dev server
pnpm build        # Next.js build
pnpm check-types  # next typegen && tsc --noEmit
pnpm lint         # Biome check
npx orval         # Regenerate API client from Swagger spec (requires API running)
```

## API Architecture

The API follows a strict **Route → Use Case** pattern:

- **Routes** (`src/routes/`) — Handle HTTP, validate with Zod, check auth, call one use case, handle errors. No business logic.
- **Use Cases** (`src/use-cases/`) — All business logic. Classes with an `execute(dto: InputDto): Promise<OutputDto>` method. Call Prisma directly (no repositories). Throw custom errors, never catch them.
- **Schemas** (`src/schemas/index.ts`) — Shared Zod schemas for request/response validation, including `ErrorSchema`.
- **Errors** (`src/errors/index.ts`) — Custom error classes. Add new ones here when needed.
- **Auth** (`src/lib/auth.ts`) — BetterAuth setup. Use `auth.api.getSession({ headers: fromNodeHeaders(request.headers) })` to verify sessions in routes.
- **Prisma** (`src/lib/prisma.ts`) — Prisma client with pg adapter. Generated client output: `src/generated/prisma/`.

**Key constraints:**
- Use Cases must map Prisma results to `OutputDto` — never return Prisma models directly.
- Use `z.enum(WeekDay)` from `../generated/prisma/enums.js` for week day fields, never `z.string()`.
- Always include `tags` and `summary` in route schemas for Swagger docs.
- API docs available at `/docs` when server is running (port 8080).

## Web Architecture

**Next.js App Router** with server-first data fetching:

- **Data fetching** — Prefer Server Components. Use functions from `src/lib/api/fetch-generated/index.ts` for server-side fetching (generated by Orval with `customFetch` mutator that forwards cookies).
- **Client-side fetching** — Use hooks from `src/lib/api/fetch-generated/index.ts` (Orval-generated). Use `mutate` (sync) variant, not `mutateAsync`, and handle success/error via `onSuccess`/`onError` callbacks.
- **Auth** — BetterAuth via `authClient` (`src/lib/auth-client.ts`). Never use middleware for auth checks — always verify in the page with `authClient.useSession()`. Never wrap `authClient` calls in try/catch; destructure `error` from the result.
- **Forms** — React Hook Form + Zod + shadcn/ui `Form` component from `@/components/ui/form`.
- **Styling** — Tailwind CSS v4 with shadcn/ui. Never use hardcoded color classes (e.g. `text-white`, `bg-black`). Always use theme tokens (e.g. `text-foreground`, `bg-primary`). Custom colors defined in `src/app/globals.css`.
- **Components** — shadcn/ui first. One component per file. Always use `Button` from `@/components/ui/button`, never native `<button>`. Always use `Image` from Next.js.

**Orval code generation:** Run `npx orval` from `apps/web` with the API running. It reads the OpenAPI spec from `$NEXT_PUBLIC_API_URL/swagger.json` and regenerates `src/lib/api/fetch-generated/index.ts`.

## TypeScript Conventions (both apps)

- Named exports over default exports (except where required, e.g. Next.js pages)
- Arrow functions over regular functions
- `interface` over `type` (unless a type alias is strictly necessary)
- kebab-case for file names, PascalCase for classes, camelCase for variables/functions
- Never use `any`
- Always use `dayjs` for date manipulation and formatting
- Always use Zod v4 (not v3). Use `z.iso.date()`, `z.iso.datetime()` etc. for ISO date validation
- Prefer early returns over nested ifs
- Receive more than 2 params as an object
